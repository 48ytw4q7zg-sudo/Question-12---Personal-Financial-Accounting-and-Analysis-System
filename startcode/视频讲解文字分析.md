```java
// library-backend/src/main/java/com/library/interceptor/LoginInterceptor.java

public class LoginInterceptor implements HandlerInterceptor {

    @Override
    public boolean preHandle(HttpServletRequest request,
                             HttpServletResponse response,
                             Object handler) throws Exception {

        // 放行 OPTIONS 请求（前端跨域预检）
        if ("OPTIONS".equalsIgnoreCase(request.getMethod())) {
            return true;
        }

        // 从请求头获取 Token
        String token = request.getHeader("Authorization");
        if (token != null && token.startsWith("Bearer ")) {
            token = token.substring(7);                       // 去掉 "Bearer " 前缀

            if (JwtUtils.validateToken(token)) {
                Claims claims = JwtUtils.parseToken(token);
                request.setAttribute("userId",   claims.get("userId"));   // ← 光标在此
                request.setAttribute("username", claims.get("username"));
                request.setAttribute("role",     claims.get("role"));
                return true;                                  // 验证通过，放行
            }
        }

        // Token 无效，返回 401
        response.setContentType("application/json;charset=UTF-8");
        response.setStatus(401);
        response.getWriter().write(
            new ObjectMapper().writeValueAsString(/* Result.fail(401, "未登录") */)
        );
        return false;
    }
}
```

**讲解要点（推断）**：

- 后端拦截器解析前端发来的 `Bearer <token>`，调用 JwtUtils 验证 Token 有效性
- 验证通过后将 userId、username、role 写入 request attribute，供后续 Controller 使用 `@RequestAttribute` 获取
- 验证失败返回 401，前端响应拦截器会捕获此状态并自动跳转到登录页

```javascript
// 响应拦截器
request.interceptors.response.use(
  // ── 成功响应（HTTP 2xx）──
  (response) => {
    const res = response.data;
    if (res.code === 200) {           // ← 光标在此，高亮选中
      return res;                     // 业务成功，返回数据
    } else {
      ElMessage.error(res.message || "操作失败");
      return Promise.reject(new Error(res.message));
    }
  },

  // ── 请求错误（网络错误、HTTP 4xx/5xx）──
  (error) => {
    if (error.response?.status === 401) {
      // Token 过期或无效：清除本地存储，跳转登录页
      localStorage.removeItem("token");
      localStorage.removeItem("userInfo");
      const currentPath = router.currentRoute.value.path;
      if (currentPath !== "/login") {
        router.push("/login");        // 避免重复跳转
      }
    } else {
      ElMessage.error(error.message || "操作失败");
    }
    return Promise.reject(error.message);
  }
);

export default request;
```

**讲解要点（推断）**：

- 响应拦截器负责统一处理业务错误（`code !== 200`）和 HTTP 错误

```
BorrowController.applyBorrow()
    ↓ @RequestAttribute("userId") 直接获取用户 ID
    ↓ 调用 Service 层执行业务逻辑
    ↓
返回 Result 响应
    ↓ index.js 响应拦截器
    ↓ res.code === 200 → 返回数据
    ↓ 401 → 清除 token，跳转 /login
```

### 2. 借阅申请的 7 步校验逻辑（BorrowServiceImpl.applyBorrow）

```
① 用户状态校验   → user.status == 1（正常用户，未被禁用）
② 图书存在校验   → book != null
③ 图书状态校验   → book.status == 1（已上架）
④ 库存校验       → book.stock > 0
⑤ 借阅数量上限   → 用户当前借阅数 < 系统上限
⑥ 最大借阅数量   → 上限为 5 本
⑦ 重复借阅校验   → 该用户对该图书无 pending/borrowing 中的记录
      ↓
所有校验通过 → 创建 BorrowRecord（status="pending"）
```
