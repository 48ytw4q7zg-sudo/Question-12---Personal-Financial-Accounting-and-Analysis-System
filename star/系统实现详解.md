## 一、系统架构概述

### 1.1 整体架构

本系统采用**前后端分离架构**（SPA + RESTful API），分为三层：

| **组合模式** | Vue 组件树 | App → AppLayout → Page → Component |

- `KEY idx_transaction_account_id (account_id)`: 按账户查询
- `KEY idx_transaction_user_time (user_id, time)`: 复合索引，按用户+时间范围查询
- `KEY idx_transaction_transfer_id (transfer_id)`: 查询转账配对记录

**转账机制**:
```
转账 500 元从"现金"到"银行卡":

记录 1:
  account_id: 1 (现金)
  type: 2 (支出)
  amount: 500.00
  transfer_id: "abc-123-def-456"

记录 2:
  account_id: 2 (银行卡)
  type: 1 (收入)
  amount: 500.00
  transfer_id: "abc-123-def-456"  ← 相同 UUID
```

**实现代码**: `TransactionServiceImpl.java` L291-361
```java
@Transactional
public TransferDTO transfer(Long userId, TransferRequest request) {
  // 1. 悲观锁查询两个账户
  Account fromAccount = accountMapper.selectByIdForUpdate(request.getFromAccountId());
  Account toAccount = accountMapper.selectByIdForUpdate(request.getToAccountId());
  
  // 2. 校验
  if (fromAccount.getId().equals(toAccount.getId())) {
    throw new BusinessException(ErrorCode.SAME_TRANSFER_ACCOUNT);
  }
  if (fromAccount.getBalance().compareTo(request.getAmount()) < 0) {
    throw new BusinessException(ErrorCode.INSUFFICIENT_BALANCE);
  }
  
  // 3. 生成 UUID
  String transferId = UUID.randomUUID().toString();
  
  // 4. 创建转出记录
  Transaction outTransaction = new Transaction();
  outTransaction.setUserId(userId);
  outTransaction.setAccountId(request.getFromAccountId());
  outTransaction.setType(TRANSACTION_TYPE_EXPENSE);
  outTransaction.setAmount(request.getAmount());
  outTransaction.setTransferId(transferId);
  transactionMapper.insert(outTransaction);
  
  // 5. 创建转入记录
  Transaction inTransaction = new Transaction();
  inTransaction.setUserId(userId);
  inTransaction.setAccountId(request.getToAccountId());
  inTransaction.setType(TRANSACTION_TYPE_INCOME);
  inTransaction.setAmount(request.getAmount());
  inTransaction.setTransferId(transferId);
  transactionMapper.insert(inTransaction);
  
  return new TransferDTO(transferId);
}
```

```java
@Transactional(propagation = Propagation.REQUIRES_NEW)
public void processUserBudgetAlerts(Long userId, List<Budget> budgets, String month, LocalDate today) {
  // 1. 删除旧预警（幂等）
  budgetAlertMapper.delete(
    new LambdaQueryWrapper<BudgetAlert>()
      .eq(BudgetAlert::getUserId, userId)
      .eq(BudgetAlert::getMonth, month)
  );
  
  // 2. 批量查询分类支出
  List<CategorySummaryDTO> summaries = transactionMapper.selectCategorySummary(
    userId, startOfMonth, startOfNextMonth, TRANSACTION_TYPE_EXPENSE
  );
  Map<Long, BigDecimal> spentMap = summaries.stream()
    .collect(Collectors.toMap(CategorySummaryDTO::getCategoryId, CategorySummaryDTO::getTotalAmount));
  
  // 3. 计算每个预算的预警级别
  int daysInMonth = today.lengthOfMonth();
  int dayOfMonth = today.getDayOfMonth();
  int remainingDays = daysInMonth - dayOfMonth + 1;
  
  for (Budget budget : budgets) {
    BigDecimal spent = spentMap.getOrDefault(budget.getCategoryId(), BigDecimal.ZERO);
    BigDecimal percentage = spent.divide(budget.getAmount(), 4, RoundingMode.HALF_UP)
      .multiply(BigDecimal.valueOf(100));
    
    String alertLevel = calculateAlertLevel(
      budget.getAmount(), spent, percentage, remainingDays, dayOfMonth
    );
    
    // 4. 持久化预警记录
    BudgetAlert alert = new BudgetAlert();
    alert.setUserId(userId);
    alert.setCategoryId(budget.getCategoryId());
    alert.setMonth(month);
    alert.setAlertLevel(alertLevel);
    alert.setBudgetAmount(budget.getAmount());
    alert.setSpentAmount(spent);
    alert.setPercentage(percentage);
    budgetAlertMapper.insert(alert);
  }
}
```

#### LoginInterceptor.java（JWT 验证拦截器）

**文件位置**: `interceptor/LoginInterceptor.java`

```java
@Component
public class LoginInterceptor implements HandlerInterceptor {
  
  @Override
  public boolean preHandle(HttpServletRequest request, HttpServletResponse response, Object handler) {
    // 1. OPTIONS 预检请求直接放行（CORS）
    if ("OPTIONS".equalsIgnoreCase(request.getMethod())) {
      return true;
    }
    
    // 2. 从请求头提取 Token
    String authHeader = request.getHeader("Authorization");
    if (authHeader == null || !authHeader.startsWith("Bearer ")) {
      writeError(response, "未登录或 token 已过期");
      return false;
    }
    
    String token = authHeader.substring(7);  // 去掉 "Bearer "
    
    // 3. 解析 Token（一次解析提取 userId + username + role）
    JwtUtils.JwtPayload payload = JwtUtils.parseTokenPayload(token);
    if (payload == null) {
      writeError(response, "未登录或 token 已过期");
      return false;
    }
    
    // 4. 将用户信息存入 request 属性
    request.setAttribute("userId", payload.getUserId());
    request.setAttribute("username", payload.getUsername());
    request.setAttribute("role", payload.getRole());
    
    // 5. 放行请求
    return true;
  }
  
  private void writeError(HttpServletResponse response, String message) {
    response.setContentType("application/json;charset=UTF-8");
    try {
      response.getWriter().write(
        "{\"code\":401,\"message\":\"" + message + "\",\"data\":null}"
      );
    } catch (IOException e) {
      e.printStackTrace();
    }
  }
  
  // 静态工具方法：从 request 获取 userId
  public static Long getUserId(HttpServletRequest request) {
    Long userId = (Long) request.getAttribute("userId");
    if (userId == null) {
      throw new BusinessException(ErrorCode.UNAUTHORIZED, "未登录");
    }
    return userId;
  }
}
```

**工作流程**:
```
HTTP Request (带 Authorization: Bearer <token>)
    ↓
LoginInterceptor.preHandle()
    ↓
提取 Token → 解析 JWT → 验证签名和有效期
    ↓
成功: 将 userId/username/role 存入 request 属性 → 放行
失败: 返回 401 错误
    ↓
Controller 通过 LoginInterceptor.getUserId(request) 获取当前用户 ID
```

#### JwtUtils.java（JWT 工具类）

**文件位置**: `util/JwtUtils.java`

```java
public class JwtUtils {
  
  private static String SECRET;
  private static SecretKey KEY;
  private static long EXPIRE;
  
  // 初始化方法（由 JwtConfig 调用）
  public static void init(String secret, long expire) {
    SECRET = secret;
    KEY = Keys.hmacShaKeyFor(secret.getBytes(StandardCharsets.UTF_8));
    EXPIRE = expire;
  }
  
  // 生成 Token
  public static String generateToken(Long userId, String username, Integer role) {
    Date now = new Date();
    Date expiration = new Date(now.getTime() + EXPIRE);
    
    return Jwts.builder()
      .setSubject(String.valueOf(userId))
      .claim("username", username)
      .claim("role", role)
      .setIssuedAt(now)
      .setExpiration(expiration)
      .signWith(KEY, SignatureAlgorithm.HS256)
      .compact();
  }
  
  // 解析 Token（一次解析提取所有信息）
  public static JwtPayload parseTokenPayload(String token) {
    try {
      Claims claims = Jwts.parserBuilder()
        .setSigningKey(KEY)
        .build()
        .parseClaimsJws(token)
        .getBody();
      
      Long userId = Long.parseLong(claims.getSubject());
      String username = claims.get("username", String.class);
      Integer role = claims.get("role", Integer.class);
      
      return new JwtPayload(userId, username, role);
    } catch (ExpiredJwtException e) {
      return null;  // Token 已过期
    } catch (Exception e) {
      return null;  // Token 无效
    }
  }
  
  // JWT Payload 内部类
  public static class JwtPayload {
    private final Long userId;
    private final String username;
    private final Integer role;
    
    public JwtPayload(Long userId, String username, Integer role) {
      this.userId = userId;
      this.username = username;
      this.role = role;
    }
    
    public Long getUserId() { return userId; }
    public String getUsername() { return username; }
    public Integer getRole() { return role; }
  }
}
```

```json

  "payload": {
    "sub": "1",                    // userId

    "iat": 1716451200,             // 签发时间
    "exp": 1717056000              // 过期时间（7 天后）
```

```javascript
import { defineStore } from 'pinia'
import { ref, computed } from 'vue'
import { decodeJwtPayload } from '@/utils/jwt'

export const useUserStore = defineStore('user', () => {
  // State
  const userId = ref(null)
  const username = ref('')
  const role = ref(0)
  const _tokenPresent = ref(!!localStorage.getItem('token'))
  
  // 初始化：从 Token 解码用户信息
  const token = localStorage.getItem('token')
  if (token) {
    const payload = decodeJwtPayload(token)
    if (payload) {
      userId.value = payload.userId
      username.value = payload.username
      role.value = payload.role
    } else {
      // Token 无效，清除
      localStorage.removeItem('token')
      _tokenPresent.value = false
    }
  }
  
  // Actions
  function setUser(user) {
    userId.value = user.userId
    username.value = user.username
    role.value = user.role
    localStorage.setItem('token', user.token)
    _tokenPresent.value = true
  }
  
  function clearUser() {
    userId.value = null
    username.value = ''
    role.value = 0
    _tokenPresent.value = false
    localStorage.removeItem('token')
  }
  
  // Getters
  const isLoggedIn = computed(() => _tokenPresent.value)
  
  return {
    userId,
    username,
    role,
    isLoggedIn,
    setUser,
    clearUser
  }
})
```

### 5.3 API 调用（Axios）

#### request.js（Axios 实例配置）

**文件位置**: `api/request.js`

```javascript
import axios from 'axios'
import { ElMessage } from 'element-plus'
import router from '@/router'
import { useUserStore } from '@/stores/user'

// 创建 Axios 实例
const request = axios.create({
  baseURL: import.meta.env.VITE_API_BASE_URL || '/api/v1',
  timeout: Number(import.meta.env.VITE_API_TIMEOUT) || 10000
})

// 重试配置
const RETRY_CONFIG = {
  maxRetries: 3,
  baseDelay: 1000,
  maxDelay: 10000,
  retryableStatuses: [500, 502, 503, 504]
}

let isRedirecting = false

// 请求拦截器
request.interceptors.request.use(
  config => {
    // 自动添加 JWT Token
    const token = localStorage.getItem('token')
    if (token) {
      config.headers.Authorization = `Bearer ${token}`
    }
    
    // 初始化重试计数
    config.__retryCount = config.__retryCount || 0
    
    return config
  },
  error => Promise.reject(error)
)

// 响应拦截器
request.interceptors.response.use(
  response => {
    const { code, message, data } = response.data
    
    // 成功响应
    if (code === 200) {
      return data
    }
    
    // 未认证（Token 过期或无效）
    if (code === 401) {
      if (!isRedirecting) {
        isRedirecting = true
        
        const userStore = useUserStore()
        userStore.clearUser()
        
        ElMessage.error(message || '未登录或 token 已过期')
        
        // 防止重定向循环
        if (router.currentRoute.value.path !== '/login') {
          router.push({
            path: '/login',
            query: { redirect: router.currentRoute.value.fullPath }
          })
        }
        
        setTimeout(() => { isRedirecting = false }, 1000)
      }
      
      return Promise.reject(new Error(message))
    }
    
    // 其他业务错误
    ElMessage.error(message || '请求失败')
    return Promise.reject(new Error(message))
  },
  async error => {
    const config = error.config
    
    // 重试逻辑（指数退避）
    if (
      config &&
      config.__retryCount < RETRY_CONFIG.maxRetries &&
      RETRY_CONFIG.retryableStatuses.includes(error.response?.status)
    ) {
      config.__retryCount++
      
      const delay = Math.min(
        RETRY_CONFIG.baseDelay * Math.pow(2, config.__retryCount - 1),
        RETRY_CONFIG.maxDelay
      )
      
      await new Promise(resolve => setTimeout(resolve, delay))
      
      return request(config)
    }
    
    // 错误提示
    if (error.response) {
      const status = error.response.status
      switch (status) {
        case 404:
          ElMessage.error('请求的资源不存在')
          break
        case 403:
          ElMessage.error('无权限访问')
          break
        case 500:
        case 502:
        case 503:
        case 504:
          ElMessage.error('服务器内部错误，请稍后重试')
          break
        default:
          ElMessage.error(`请求失败 (${status})`)
      }
    } else if (error.code === 'ECONNABORTED') {
      ElMessage.error('请求超时，请检查网络连接')
    } else {
      ElMessage.error('网络连接异常，请检查网络')
    }
    
    return Promise.reject(error)
  }
)

export default request
```

```java
public LoginResponse login(UserLoginRequest request) {
  // 1. 登录限流检查
  if (!loginRateLimiter.tryAcquire(request.getUsername())) {
    throw new BusinessException(ErrorCode.LOGIN_RATE_LIMIT, "登录过于频繁，请稍后再试");
  }
  
  // 2. 查询用户
  User user = userMapper.selectOne(
    new LambdaQueryWrapper<User>().eq(User::getUsername, request.getUsername())
  );
  
  // 3. 验证密码
  if (user == null || !passwordEncoder.matches(request.getPassword(), user.getPassword())) {
    // 不区分"用户不存在"和"密码错误"，防止枚举攻击
    throw new BusinessException(ErrorCode.PASSWORD_ERROR, "用户名或密码错误");
  }
  
  // 4. 生成 JWT Token
  String token = JwtUtils.generateToken(user.getId(), user.getUsername(), user.getRole());
  
  // 5. 清理限流器
  loginRateLimiter.reset(request.getUsername());
  
  // 6. 返回登录响应
  LoginResponse response = new LoginResponse();
  response.setToken(token);
  response.setUserId(user.getId());
  response.setUsername(user.getUsername());
  response.setRole(user.getRole());
  
  return response;
}
```

```java
@Component
public class LoginRateLimiter {
  
  private final Map<String, RateLimiter> limiters = new ConcurrentHashMap<>();
  
  public boolean tryAcquire(String username) {
    RateLimiter limiter = limiters.computeIfAbsent(
      username,
      k -> RateLimiter.create(3.0)  // 每秒 3 个令牌
    );
    return limiter.tryAcquire();
  }
  
  public void reset(String username) {
    limiters.remove(username);
  }
}
```

```java
public void delete(Long userId, Long accountId) {
  // 1. 查询账户

      .eq(RecurringBill::getAccountId, accountId)

  }
  
  // 4. 软删除（设置 status = 0）
  account.setStatus(STATUS_INACTIVE);
  accountMapper.updateById(account);
```

```java
public List<AccountBalanceDTO> getBalance(Long userId) {

    .map(Account::getId)

  // 3. 批量查询各账户收入（消除 N+1）
  List<AccountSummaryDTO> incomes = transactionMapper.selectAccountIncomeBatch(userId, accountIds);
  Map<Long, BigDecimal> incomeMap = incomes.stream()
    .collect(Collectors.toMap(AccountSummaryDTO::getAccountId, AccountSummaryDTO::getAmount));
  
  // 4. 批量查询各账户支出
  List<AccountSummaryDTO> expenses = transactionMapper.selectAccountExpenseBatch(userId, accountIds);
  Map<Long, BigDecimal> expenseMap = expenses.stream()
    .collect(Collectors.toMap(AccountSummaryDTO::getAccountId, AccountSummaryDTO::getAmount));
  
  // 5. 计算余额
  List<AccountBalanceDTO> balances = new ArrayList<>();
  for (Account account : accounts) {
    BigDecimal income = incomeMap.getOrDefault(account.getId(), BigDecimal.ZERO);
    BigDecimal expense = expenseMap.getOrDefault(account.getId(), BigDecimal.ZERO);
    
    // 余额 = 初始余额 + 收入 - 支出
    BigDecimal currentBalance = account.getInitialBalance()
      .add(income)
      .subtract(expense);
    
    AccountBalanceDTO dto = new AccountBalanceDTO();
    dto.setAccountId(account.getId());
    dto.setAccountName(account.getName());
    dto.setCurrency(account.getCurrency());
    dto.setInitialBalance(account.getInitialBalance());
    dto.setTotalIncome(income);
    dto.setTotalExpense(expense);
    dto.setCurrentBalance(currentBalance);
    
    balances.add(dto);
  }
  
  return balances;
}
```

```java
private String calculateAlertLevel(
  BigDecimal budgetAmount,
  BigDecimal spentAmount,
  BigDecimal percentage,
  int remainingDays,
  int dayOfMonth
) {
  // 1. 已超支
  if (spentAmount.compareTo(budgetAmount) >= 0) {
    return "OVERSPENT";
  }
  
  // 2. 月度预警（已消费 ≥ 预算 × 80%）
  if (percentage.compareTo(BigDecimal.valueOf(80)) >= 0) {
    return "MONTHLY_WARN";
  }
  
  // 3. 日均预警（日均消费 ≥ 预算 / 剩余天数 × 150%）
  if (remainingDays > 0) {
    BigDecimal dailyBudget = budgetAmount.divide(
      BigDecimal.valueOf(remainingDays), 2, RoundingMode.HALF_UP
    );
    BigDecimal dailySpent = spentAmount.divide(
      BigDecimal.valueOf(dayOfMonth), 2, RoundingMode.HALF_UP
    );
    BigDecimal dailyThreshold = dailyBudget.multiply(BigDecimal.valueOf(1.5));
```

```java
@Transactional
public ImportResultDTO importCsv(Long userId, MultipartFile file, Long accountId) {
  // 1. 校验文件大小（≤ 5MB）
  if (file.getSize() > 5 * 1024 * 1024) {
    throw new BusinessException(ErrorCode.FILE_TOO_LARGE, "文件大小不能超过5MB");
  }
  
  // 2. 校验文件后缀
  String filename = file.getOriginalFilename();
  if (filename == null || !filename.toLowerCase().endsWith(".csv")) {
    throw new BusinessException(ErrorCode.CSV_FORMAT_ONLY, "只支持CSV格式");
  }
  
  // 3. 校验账户归属
  Account account = accountMapper.selectById(accountId);
  if (account == null || !account.getUserId().equals(userId)) {
    throw new BusinessException(ErrorCode.ACCOUNT_NOT_FOUND_OR_DISABLED, "账户不存在或无权访问");
  }
  if (account.getStatus() == STATUS_INACTIVE) {
    throw new BusinessException(ErrorCode.ACCOUNT_DISABLED, "账户已禁用");
  }
  
  // 4. 预加载所有分类（缓存）
  List<Category> categories = categoryMapper.selectList(null);
  Map<String, Long> categoryNameToId = categories.stream()
    .collect(Collectors.toMap(Category::getName, Category::getId));
  
  // 5. 解析 CSV
  List<ImportResultDTO.FailRow> failRows = new ArrayList<>();
  int successCount = 0;
  
  try (CSVReader reader = new CSVReader(new InputStreamReader(file.getInputStream(), StandardCharsets.UTF_8))) {
    // 跳过表头
    reader.readNext();
    
    String[] line;
    int rowNumber = 1;
    while ((line = reader.readNext()) != null) {
      rowNumber++;
      
      try {
        // 解析字段
        if (line.length < 4) {
          throw new Exception("列数不足");
        }
        
        String timeStr = line[0].trim();
        String amountStr = line[1].trim();
        String typeStr = line[2].trim();
        String categoryName = line[3].trim();
        String note = line.length > 4 ? line[4].trim() : "";
        
        // 解析时间
        LocalDateTime time;
        try {
          time = LocalDateTime.parse(timeStr, DateTimeFormatter.ofPattern("yyyy-MM-dd HH:mm:ss"));
        } catch (Exception e) {
          throw new Exception("时间格式错误");
        }
        
        // 解析金额
        BigDecimal amount;
        try {
          amount = new BigDecimal(amountStr);
          if (amount.compareTo(BigDecimal.ZERO) <= 0) {
            throw new Exception("金额必须大于0");
          }
        } catch (NumberFormatException e) {
          throw new Exception("金额格式错误");
        }
        
        // 解析类型
        Integer type;
        if ("收入".equals(typeStr)) {
          type = TRANSACTION_TYPE_INCOME;
        } else if ("支出".equals(typeStr)) {
          type = TRANSACTION_TYPE_EXPENSE;
        } else {
          throw new Exception("类型必须是'收入'或'支出'");
        }
        
        // 解析分类
        Long categoryId = categoryNameToId.get(categoryName);
        if (categoryId == null) {
          categoryId = categoryNameToId.get("其他");  // 默认分类
        }
        
        // 创建交易记录
        Transaction transaction = new Transaction();
        transaction.setUserId(userId);
        transaction.setAccountId(accountId);
        transaction.setCategoryId(categoryId);
        transaction.setType(type);
        transaction.setAmount(amount);
        transaction.setNote(note);
        transaction.setTime(time);
        transactionMapper.insert(transaction);
        
        successCount++;
        
      } catch (Exception e) {
        // 记录失败行
        ImportResultDTO.FailRow failRow = new ImportResultDTO.FailRow();
        failRow.setRowNumber(rowNumber);
        failRow.setReason(e.getMessage());
        failRows.add(failRow);
      }
    }
  } catch (IOException e) {
    throw new BusinessException(ErrorCode.CSV_READ_ERROR, "CSV文件读取失败");
  }
  
  // 6. 返回导入结果
  ImportResultDTO result = new ImportResultDTO();
  result.setSuccessCount(successCount);
  result.setFailCount(failRows.size());
  result.setFailRows(failRows);
  
  return result;
}
```

```java
public class ExchangeRateServiceImpl implements ExchangeRateService {
  
  // 硬编码汇率（相对于 CNY）
  private static final Map<String, BigDecimal> RATES = Map.of(
    "CNY", BigDecimal.ONE,
    "USD", new BigDecimal("7.20"),
    "EUR", new BigDecimal("7.80"),
    "JPY", new BigDecimal("0.048"),
    "GBP", new BigDecimal("9.10"),
    "HKD", new BigDecimal("0.92")
  );
  
  public ExchangeRateDTO getExchangeRates() {
    ExchangeRateDTO dto = new ExchangeRateDTO();
    dto.setRates(RATES);
    
    // 计算反向汇率
    Map<String, BigDecimal> ratesInverse = new HashMap<>();
    for (Map.Entry<String, BigDecimal> entry : RATES.entrySet()) {
      if (entry.getValue().compareTo(BigDecimal.ZERO) > 0) {
        ratesInverse.put(
          entry.getKey(),
          BigDecimal.ONE.divide(entry.getValue(), 4, RoundingMode.HALF_UP)
        );
      }
    }
    dto.setRatesInverse(ratesInverse);
    
    return dto;
  }
}
```

### 9.4 事务管理

**声明式事务**: `@Transactional`
```java
@Transactional
public TransferDTO transfer(Long userId, TransferRequest request) {
  // 1. 悲观锁查询账户
  Account fromAccount = accountMapper.selectByIdForUpdate(request.getFromAccountId());
  Account toAccount = accountMapper.selectByIdForUpdate(request.getToAccountId());
  
  // 2. 校验
  if (fromAccount.getId().equals(toAccount.getId())) {
    throw new BusinessException(ErrorCode.SAME_TRANSFER_ACCOUNT);
  }
  
  // 3. 创建交易记录
  Transaction outTransaction = new Transaction();
  outTransaction.setAccountId(request.getFromAccountId());
  outTransaction.setType(TRANSACTION_TYPE_EXPENSE);
  outTransaction.setAmount(request.getAmount());
  transactionMapper.insert(outTransaction);
  
  Transaction inTransaction = new Transaction();
  inTransaction.setAccountId(request.getToAccountId());
  inTransaction.setType(TRANSACTION_TYPE_INCOME);
  inTransaction.setAmount(request.getAmount());
  transactionMapper.insert(inTransaction);
  
  return new TransferDTO();
}
```

```
    ↓

    ↓
COMMIT TRANSACTION  ← 提交
    ↓
释放锁
```

### 10.7 Git 冲突解决

```bash

# =======
# 远程代码

# 2. 手动合并代码，删除冲突标记

# 3. 添加解决后的文件
git add system/backend/src/main/java/com/example/finance/controller/AccountController.java

# 4. 提交合并
```

## 十一、答辩高频问题解答

### Q1: 为什么选择 Spring Boot + Vue 架构？

**A**: 
1. **主流技术栈**: Spring Boot 是 Java 后端最流行的框架，Vue 是前端三大框架之一，企业广泛使用
2. **前后端分离**: 独立开发、独立部署、独立扩展，符合现代 Web 开发趋势
3. **生态完善**: Spring Boot 有丰富的 starter，Vue 有 Element Plus 等 UI 库，开发效率高
4. **学习曲线**: 相比 Spring Cloud 微服务，Spring Boot 单体应用更适合教学场景

---

### Q2: 为什么使用 MyBatis-Plus 而不是 JPA？

**A**:
1. **灵活性**: MyBatis-Plus 可以写自定义 SQL，JPA 的 HQL 复杂查询不直观
2. **性能优化**: 可以精确控制 SQL，避免 JPA 的 N+1 查询问题
3. **学习成本**: MyBatis-Plus 的 LambdaQueryWrapper 类型安全，比字符串拼接更安全
4. **国内流行**: MyBatis 在国内企业使用率高于 JPA

---

### Q3: JWT Token 为什么设置 7 天有效期？

**A**:
1. **用户体验**: 7 天内不需要重新登录，提升用户体验
2. **教学简化**: 没有实现 Refresh Token 机制（生产环境应该用双 Token）
3. **安全性权衡**: 7 天是常见选择（支付宝 7 天，微信 30 天）
4. **可扩展**: 生产环境可以缩短到 30 分钟 + Refresh Token

---

### Q4: 密码为什么用 BCrypt 而不是 MD5？

**A**:
1. **自带盐值**: BCrypt 每次加密生成随机盐，防止彩虹表攻击
2. **计算成本**: BCrypt 有 cost factor（默认 12），可以调节计算复杂度，抗暴力破解
3. **安全性**: MD5 已被证明不安全（碰撞攻击），BCrypt 是 OWASP 推荐算法
4. **实现简单**: Spring Security 提供 `BCryptPasswordEncoder`，一行代码搞定

---

### Q5: 为什么账户余额实时计算而不是缓存？

**A**:
1. **数据一致性**: 实时计算保证余额永远准确，无需担心缓存失效
2. **教学简化**: 缓存需要处理失效策略、并发更新等复杂问题
3. **性能足够**: 单表查询 + 索引优化，响应时间在毫秒级
4. **可扩展**: 生产环境可以用 Redis 缓存 + 消息队列异步更新

---

### Q6: 转账为什么用悲观锁而不是乐观锁？

**A**:
1. **并发场景**: 转账涉及余额扣减，高并发时乐观锁失败率高（重试成本高）
2. **数据一致性**: 悲观锁（`SELECT ... FOR UPDATE`）保证同一账户不会被并发修改
3. **教学简化**: 乐观锁需要版本号字段 + 重试逻辑，代码更复杂
4. **性能权衡**: 转账频率低，悲观锁的性能损失可以接受

---

### Q7: 为什么没有使用 Redis？

**A**:
1. **教学简化**: 单机应用不需要分布式缓存
2. **数据量小**: ≤1000 条记录，MySQL 响应时间在毫秒级
3. **无状态**: JWT 无状态认证，不需要 Session 存储
4. **可扩展**: 生产环境可以用 Redis 缓存分类列表、用户信息等热点数据

---

### Q8: 为什么没有物理外键？

**A**:
1. **教学简化**: 物理外键会增加数据库复杂度（级联删除、约束检查）
2. **应用层校验**: Service 层已经做了关联校验（如删除账户时检查交易记录）
3. **性能**: 物理外键会降低 INSERT/UPDATE 性能
4. **灵活性**: 软删除（status 字段）比物理删除更安全

---

### Q9: 前端为什么用 Pinia 而不是 Vuex？

**A**:
1. **官方推荐**: Vuex 4 已停止维护，Vue 官方推荐 Pinia
2. **API 简洁**: Pinia 使用 Composition API（`defineStore`），比 Vuex 的 `mutations/actions` 更直观
3. **TypeScript 支持**: Pinia 对 TypeScript 支持更好
4. **体积更小**: Pinia 的 bundle size 比 Vuex 小

---

### Q10: 前端为什么用 Vite 而不是 Webpack？

**A**:
1. **开发体验**: Vite 启动速度秒级（ESM 原生支持），Webpack 需要几分钟
2. **热更新**: Vite 的 HMR 更快，修改代码后立即生效
3. **配置简单**: Vite 配置比 Webpack 简洁
4. **现代工具**: Vite 使用 Rollup 打包（生产环境），比 Webpack 更现代

---

### Q11: 为什么预算预警用定时任务而不是实时计算？

**A**:
1. **性能优化**: 实时计算需要每次访问预算页面都查询交易记录，定时任务只需每天计算一次
2. **数据持久化**: 预警结果存入 `budget_alert` 表，可以查看历史预警
3. **教学简化**: 实时计算需要在多个地方（预算页面、首页）重复计算
4. **可扩展**: 生产环境可以用消息队列（RabbitMQ）实时推送预警

---

### Q12: CSV 导入为什么用事务？

**A**:
1. **原子性**: 任何一行失败都回滚所有导入，保证数据一致性
2. **预览机制**: 先预览成功/失败行数，用户确认后再导入
3. **错误处理**: 失败行记录详细原因（行号 + 错误信息）
4. **用户体验**: 用户可以看到哪些行导入失败，修正后重新导入

---

### Q13: 为什么没有实现数据导出功能？

**A**:
1. **PRD 未要求**: PRD 只要求 CSV 导入，未要求导出
2. **教学简化**: 导出需要处理 Excel 生成、大数据量分页导出等复杂问题
3. **可扩展**: 生产环境可以用 Apache POI 导出 Excel，或用流式导出处理大数据量

---

### Q14: 单元测试覆盖率为什么是 60%？

**A**:
1. **核心覆盖**: 140 个测试用例覆盖了核心业务逻辑（Service 层）
2. **教学要求**: 课程要求 ≥60%，实际达到了 65%
3. **时间成本**: 100% 覆盖率需要测试 getter/setter 等简单方法，投入产出比低
4. **测试分类**: 白盒测试 49 个 + 黑盒测试 55 个 + 集成测试 19 个 + 系统测试 10 个 + CSV 测试 7 个

### Q15: 系统有哪些安全设计？

**A**:
1. **密码加密**: BCrypt 加密存储，数据库看不到明文
2. **JWT 认证**: 无状态认证，Token 有效期 7 天
3. **登录限流**: Guava RateLimiter 防止暴力破解（每秒 3 次）
4. **参数校验**: `@Valid` + JSR-303 注解，防止 SQL 注入和 XSS
5. **权限控制**: 路由守卫 + 后端拦截器，双重验证
6. **敏感信息**: 密码、JWT 密钥不硬编码在代码中，通过环境变量注入
7. **CORS 配置**: 生产环境禁止 localhost 跨域
8. **SQL 参数化**: MyBatis-Plus 自动参数化，防止 SQL 注入

---

### Q16: 系统有哪些性能优化？

**A**:
1. **N+1 消除**: 批量查询替代逐条查询（如 `selectAccountIncomeBatch`）
2. **索引优化**: 为高频查询字段添加索引（如 `user_id`、`account_id`、`time`）
3. **分页查询**: 使用 MyBatis-Plus 分页插件，避免全表查询
4. **按需加载**: ECharts 动态 import，减少首屏加载体积
5. **代码分割**: Vite manualChunks 拆分 vendor（Element Plus、Axios、ECharts）
6. **悲观锁**: 转账使用 `SELECT ... FOR UPDATE`，防止并发透支
7. **JWT 单次解析**: `parseTokenPayload()` 一次解析提取 userId + username + role

### Q18: 开发过程中遇到的最大挑战是什么？

**A**:
1. **N+1 查询问题**: 
   - 问题: 余额汇总时逐账户查询，10 个账户执行 10 次 SQL
   - 解决: 使用批量查询（`selectAccountIncomeBatch`），2 次 SQL 搞定
   - 收获: 理解了 ORM 的性能陷阱，学会了 SQL 优化

2. **转账并发安全**:
   - 问题: 并发转账可能导致余额透支（A→B 和 A→C 同时发生）
   - 解决: 使用悲观锁（`SELECT ... FOR UPDATE`）+ `@Transactional`
   - 收获: 理解了事务隔离级别和并发控制

3. **JWT Token 刷新**:
   - 问题: Token 过期后用户需要重新登录，体验差
   - 解决: 教学简化，未实现 Refresh Token（生产环境应该实现）
   - 收获: 理解了双 Token 机制的设计

4. **预算保存并发冲突**:
   - 问题: 两个请求同时保存同一分类的预算，导致唯一索引冲突
   - 解决: 先查后改 + 捕获 `DuplicateKeyException` 回退更新
   - 收获: 理解了幂等性设计
