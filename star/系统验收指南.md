## §0 快速启动

### 0.1 一键启动

双击 `star\一键启动.bat`，脚本自动完成：
1. 环境检测 (Java 21 / Maven 3.9 / MySQL 8.4 / Node.js 24 / pnpm 11)
2. 数据库初始化 (如需要)
3. 后端编译 + 启动 (SpringBoot :8080)
4. 前端依赖安装 + 启动 (Vite :5173)
5. 自动打开浏览器到 `http://localhost:5173`

### 0.2 开机自启

右键「**以管理员身份运行**」`star\配置开机自启.bat`，电脑重新开机后 30 秒自动启动整个系统。

### 0.3 测试账号

| 角色 | 用户名 | 密码 | 说明 |
|---|:---:|---|:---:|
| 普通用户 | `zhangsan` | `123456` | 有完整账户/交易/预算数据 |
| 管理员 | `admin` | `123456` | 可管理所有用户 |

### 0.4 手动启动 (分步)

```bash
# 步骤 1: 初始化数据库
mysql -uroot -proot < sql/01-init.sql

# 步骤 2: 启动后端 (新终端)
cd system/backend
mvn spring-boot:run

# 步骤 3: 启动前端 (新终端)
cd system/frontend
pnpm install
pnpm dev

# 步骤 4: 浏览器访问 http://localhost:5173
```

## §6 快速问题排查

| 问题 | 解决方案 |
|---|:---|
| 端口 8080 被占用 | 任务管理器关闭 java.exe 进程, 或运行: `netstat -ano \| findstr 8080` → `taskkill /F /PID <PID>` |
| 前端报 "Network Error" | 确认后端已启动 → `curl http://localhost:8080/api/v1/health` 应返回 status:UP |
| MySQL 连接失败 | 确认 MySQL 服务已启动 (services.msc → MySQL80 → 启动) |
| 数据库表不存在 | 执行: `mysql -uroot -proot < sql/01-init.sql` |
| 登录后空白页 | F12 → Console 查看错误; 确认 `pnpm install` 已执行 |
| token 过期 | 退出重新登录即可 (默认 7 天有效期) |
