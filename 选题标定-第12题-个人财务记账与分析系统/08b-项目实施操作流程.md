### §1.2  准备题目变量(用于 /init-skeleton 参数 + CLAUDE.md 占位)

6 个变量仅在两处使用:① §2.2 `/init-skeleton` 参数(全用);② §7.2 替换 CLAUDE.md 2 处占位符(题目 + 角色)。

| 变量 | 提取 | 格式约束 | 示例(第 5 题"宿舍报修与维修调度系统") |
|------|------|------|------|
| `{{题目}}` | 标定卡 §一 题名 | 中文 | 宿舍报修与维修调度系统 |
| `{{核心实体}}` | 标定卡 §一 核心实体(4-5 个) | 英文逗号分隔(提取时去除 Markdown 反引号) | user, account, category, transaction |
| `{{核心功能}}` | 标定卡 §三 P0 主要动作 3-5 个 | 中文逗号分隔 | 报修 / 派单 / 维修跟踪 |
| `{{包名}}` | 自起 | 小写英文 + 点·三段 | `com.example.repair` |
| `{{数据库名}}` | 自起 | 小写英文 + 下划线 | `repair_db` |
| `{{项目根目录名}}` | 自起 | **严格使用「学号-题号」格式**(如 `2024010101-38`),仅含数字和连字符,禁止中文/空格/下划线 | `2024010101-38` |

**强烈建议**将 6 个变量统一抄写到 **"我的选题信息.txt"** 文本文件中,后续调用命令时直接复制粘贴,避免手动输入错误。

**核心功能**从 P0 功能清单中提取 3-5 个主要动作(如 "登录注册、账户 CRUD、收支记录")。**包名**采用三段式结构(如 `com.example.finance`),全部使用小写英文字母,单词间用点分隔。**数据库名**采用小写英文字母加下划线格式(如 `finance_db`)。

#### §1.4.2  解压到工作目录

> 前置:08a §9.4 已建好 `D:\code\` 工作区(全英文路径)。

**推荐在 D 盘根目录下创建 `code` 文件夹作为统一工作区**,将项目解压到 `D:\code\<学号-题号>` 目录下。**从磁盘根目录到项目文件夹的全路径必须严格使用英文小写字母、数字、连字符和点,绝对不能包含中文、空格、括号或任何特殊字符**(如 `!@#$%^&*`)。桌面路径因包含中文用户名,会导致 Java 编译出现 `MalformedInputException` 错误,**严禁使用**。

1. `mkdir D:\code\<学号>-<题号>`(命名 = §1.3 Gitee 仓库名)
2. 用 7-Zip / WinRAR 解压 zip **内容**到该目录(不是 zip 整体)。**解压时必须选择"解压到当前文件夹"**,不要选择"解压到 `<压缩包名>/`",否则会导致 `.claude` 目录嵌套在子文件夹中,Claude Code 无法识别命令。
3. 检查 `.claude\commands\` 下应有 32 个 `.md` 文件 · 出现嵌套则把多余层移出

> ⚠️ **严禁在执行 `/init-skeleton` 命令后修改项目根目录名**。一旦生成项目骨架,所有文件路径、配置项、导入语句均已绑定原目录名,手动修改会导致数百处路径错误,AI 无法批量修复。

解压后目录树:

```
D:\code\<学号>-<题号>\
├── .claude\
│   ├── commands\          # 32 个 .md + README.md
│   └── project-status.md  # 动态状态
├── CLAUDE.md               # 项目宪法
├── AGENTS.md
├── .gitignore
├── .editorconfig
└── README.md               # zip 用法说明(可删)
```

#### [Step 4] /refactor-helper 应用修复(改结构)

接 reviewer 对话 · 不新建任务(必须看 R-XX 注释上下文)· 保持 `/model opus`。

**小步重构**:1 issue → 1 次重构 → 跑测试 → commit · 重构前先 commit 做 safety net · 不顺手改业务逻辑。

**修复优先级**:
1. R-08 高严重度安全 issue ← 最高(可被外部利用 · 必修)· commit 前缀 `fix(p7-sec)`

4. R-08 维度 8 幂等性性能子集 · Step 5 用 perf-optimizer

#### 规则 7:何时新建任务(开新对话)防 AI 串台/幻觉

对话越长 AI 越容易忘规则/串细节/编造。每条命令 prompt 自包含 · 不需要前面对话历史撑场(上一 Phase 产物已落盘 · 下一 Phase 读文件即可)。

⚠️ Claude Code CLI **没有 token / 上下文百分比实时显示** · 不能像 ChatGPT/Cursor 靠进度条判断 · 按下方**三层判据**操作。

**三层判据**:

| 层 | 判据 | 适用 |
|:--:|---|---|
| 1 | **08c 命令字典查表** | 99% 场景 · 跑任何 `/` 命令前先查 |
| 2 | **客观计数兜底** | 字典查不到时 |
| 3 | **客观异常触发** | AI 出错救急 |

**客观计数兜底**(本会话满足任一就强烈建议下个命令前重启):
1. 已连续跑 ≥5 个斜杠命令
2. 持续时间 >1 小时
3. 已切过 ≥2 次模型(`/model haiku` ↔ `/model opus`)
4. 完成一个 Phase(对照 ✅ 检查点全勾)
5. 完成一个 P0 功能的 6 步循环

**客观异常触发**(任一满足立即重启 · 不要硬扛):
1. 单次回复 >30 秒还没开始输出文字
2. 输出过程中明显变慢或卡顿
3. AI 给的代码贴进 IDE 报红色波浪线(import/变量未定义)
4. 同 bug 同 AI 改过 3 次还不对
5. 跑同命令 ≥3 次结果不一样
6. AI 引用 `docs/` 下不存在的文件名(Ctrl+F 搜不到)
7. CLAUDE.md 规则(§一-§四)说 X · AI 偏写 Y(如 §二·四 要求 LambdaQueryWrapper · AI 用原生 SQL)
