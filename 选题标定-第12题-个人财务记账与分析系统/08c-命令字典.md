# 08c · 命令字典

> 跑任何 `/xxx` 命令**之前** → Ctrl+F 搜命令名 → 看本表"调用前重启 claude?"列 → 按表操作。

## 关键约定

- **所有命令的参数区分大小写**,例如 `P0-1` 不能写成 `p0-1`,否则会导致命令执行失败。



- **生成型命令 G-xx**(/srs-writer / /db-designer / /entity-coder / /feature-coder 等):**调用前必重启 claude**(防上下文污染影响生成质量)
- **审核类命令 R-xx**(/srs-reviewer / /db-reviewer / /code-reviewer-be 等):**调用前必重启 claude · 模型保持 V4 Pro**(有 GLM key 可切 GLM 异源 · 见 08a §11.6)
- **应用修复**(`/xxx 应用修复`):**不要退出 `claude`**(需要看 reviewer 标的 issue 注释上下文 · 接审核会话继续)
- **排查类命令 D-xx**(/bug-tracer-be / /bug-tracer-fe):**不要退出 `claude`**(需要看刚才报错信息和代码上下文)
- **通用类**(/git-committer / /rules-updater / /prompt-evolver):**不限**(都行 · 倾向新建任务避免污染)

> 📌 **字典里没找到我要跑的命令?** 按 3 个原则兜底:
> - 命令名含 `-writer` / `-coder` / `-designer` / `-prototyper` → 生成型 → **必重启 claude**
> - 命令名含 `-reviewer` / `-tracer` → 审核或排查类 → 看是 R-xx 还是 D-xx · R 类必重启 claude · D 类不要
> - 命令名含 `应用修复` → 任何命令的应用修复模式 → **不要退出 `claude`**(需 reviewer 注释上下文)
